## 密碼管理器架構解說

整個專案分成五個層次，每層各司其職：
1️⃣ 進入點 main.py
程式啟動的第一步。它只做兩件事：判斷是否首次使用（需要設定主密碼），還是回頭使用（直接開登入畫面）。它是所有模組的串接起點，本身不含任何業務邏輯。
2️⃣ core/ — 安全核心（最重要的層）
這層負責所有跟安全性相關的事，分成兩個檔案：
auth.py 管主密碼的驗證。主密碼絕對不會明文儲存，只存 bcrypt hash。每次登入都是把你輸入的密碼和 hash 比對，不是比對密碼本身。
crypto.py 管加密與解密。登入成功後，用主密碼透過 PBKDF2 衍生出一把 AES 金鑰，這把金鑰只存在記憶體，程式關閉或登出就消失，硬碟上永遠找不到它。
3️⃣ data/ — 資料存取層（你負責的部分）
分成兩個職責完全不同的檔案：
vault.py 只負責檔案 I/O，讀取和寫入 vault.enc，完全不懂加密也不懂業務邏輯，就是單純的讀寫工具。
repository.py 只負責操作記憶體中的 dict，做新增、查詢、編輯、刪除。它不碰檔案，也不碰加密，傳進來什麼 dict 就操作什麼 dict，非常單純。
4️⃣ models/entry.py — 資料模型
定義一筆密碼條目長什麼樣子，用 Python dataclass 描述欄位：標題、帳號、密碼、網址、備註、分類、時間戳記。整個專案都用這個結構傳遞資料，確保欄位一致。
5️⃣ ui/ — 使用者介面層
三個畫面，彼此獨立：
login_window.py 只做一件事：輸入主密碼、驗證、成功後把金鑰傳給主畫面。
main_window.py 是使用最頻繁的畫面，顯示密碼列表、搜尋、複製密碼（30 秒後自動清除剪貼簿）、閒置 5 分鐘自動鎖定。
entry_dialog.py 是新增或編輯時彈出的對話框，內建隨機密碼產生器。
🔑 設計的核心原則

原則 體現方式
明文不落地 vault.enc 永遠是加密內容，JSON 只存在記憶體
金鑰不落地 AES 金鑰由主密碼即時衍生，不寫入任何檔案
單一職責 每個檔案只做一件事，互不干涉
層層隔離 UI 不碰加密，repository 不碰檔案，crypto 不碰業務邏輯
